# proto

Контракты сообщений между сервисами разрабатываемого нейросетевого генератора
анимаций. Подробный контекст приведён в [корневом README](../README.md) и в
тексте ВКР.

## Назначение

Так как система состоит из нескольких взаимодействующих компонентов (менеджер
пайплайнов, исполнители разных типов задач, механизм отмены), необходимо
обеспечить единообразное описание форматов задач, ответов и сопутствующих
структур данных. Если каждый сервис будет поддерживать такие структуры
независимо, возрастает риск рассогласования форматов и ошибок сериализации.

Поэтому схемы сообщений вынесены в отдельный репозиторий и описаны в формате
`protobuf`. Для каждого языка на основе этих схем генерируются собственные
парсеры и структуры данных. Это позволяет иметь единый источник правды и
централизованно поддерживать контракты взаимодействия между сервисами.

Дополнительным преимуществом `protobuf` является бинарность, что уменьшает
размер сообщения и ускоряет процесс кодирования и декодирования, а также
встроенная поддержка механизмов прямой и обратной совместимости.

## Структура

```
proto/
├── proto/worker_service/task/   # исходные .proto-схемы
│   ├── task.proto               # обёртка-oneof над всеми типами задач
│   ├── add.proto                # добавление объекта на изображение
│   ├── delete.proto             # удаление объекта с изображения
│   ├── stand.proto              # установка персонажа в заданную позу
│   ├── animate.proto            # генерация анимации по первому кадру
│   ├── rmbg.proto               # удаление фона
│   ├── norm_map.proto           # построение карты нормалей
│   └── status.proto             # ответы и сообщения об ошибках
└── python/worker_service/       # сгенерированный Python-пакет
    ├── pyproject.toml           # установка как proto-worker-service
    └── worker_service_proto/
        └── task/__init__.py     # классы Task, TaskStatus, *Completed, ...
```

Каждая `.proto`-схема описывает один тип задачи и соответствующий ему ответ
о завершении. Например, `animate.proto` содержит сообщение `Animate` с путём
до первого кадра, текстовым описанием действия, числом кадров и флагом
зацикленности, а также сообщение `AnimateCompleted` с путём до сохранённого
набора кадров.

Сообщение `Task` из `task.proto` оборачивает все типы задач в `oneof` —
именно в таком виде задачи попадают в брокер сообщений. Сообщение
`TaskStatus` аналогичным образом оборачивает успешные ответы и общее
сообщение об ошибке `Failed`.

## Использование

### Python

Сгенерированный пакет подключается как path-зависимость. В сервисе-исполнителе
он указан в `pyproject.toml`:

```toml
[project.optional-dependencies]
service = [
    "proto-worker-service",
]

[tool.uv.sources]
proto-worker-service = { path = "../proto/python/worker_service" }
```

После установки доступен импорт классов задач:

```python
from worker_service_proto.task import Task, Animate, AnimateCompleted

task = Task(animate=Animate(
    id="anim-001",
    first_frame_path="s3://assets/inputs/character.png",
    prompt="A character running in place",
    num_frames=45,
    loop=True,
))
payload = bytes(task)
```

### Другие языки

Готовые пакеты для других языков в репозитории не публикуются. Чтобы получить
их, достаточно сгенерировать код из `proto/worker_service/task/*.proto`
любым подходящим компилятором `protoc` или его обёрткой.

## Совместимость

При изменении схем следует придерживаться стандартных правил совместимости
`protobuf`: не переиспользовать номера полей, добавлять новые поля как
опциональные, не менять типы и имена существующих полей. Это позволяет
обновлять сервисы независимо без одновременного перезапуска всей системы.
