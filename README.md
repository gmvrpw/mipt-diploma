# Нейросетевой генератор анимаций для компьютерных игр

Этот репозиторий объединяет исходный код выпускной квалификационной работы
«Разработка нейросетевого генератора анимаций для компьютерных игр» (Боярников
А. А., МФТИ, ВШПИ, 2026). Подробное описание задачи, обоснование выбора моделей
и архитектурных решений приведены в файле ВКР (смотри релизы)

## О проекте

Цель работы — повышение эффективности создания анимаций для компьютерных игр
на основе нейросетевого генератора, способного создавать кадровые анимации, в
том числе зацикленные, по единственному изображению персонажа.

Конечный пайплайн состоит из трёх последовательных модулей:

1. **Генерация когерентных изображений персонажа** — модель контекстного
   изменения изображения Qwen-Image-Edit-2509 переводит персонажа в новые
   позы и состояния, сохраняя его визуальную идентичность.
2. **Генерация видеофрагмента** — модель Wan2.2 создаёт короткую анимацию по
   ключевому кадру и текстовому описанию действия.
3. **Постобработка** — отдельные модели InspyreNet и StableNormal удаляют фон
   и строят карты нормалей для каждого кадра.

Для зацикленных анимаций реализован отдельный механизм поиска цикла в
сгенерированном видеофрагменте, основанный на сравнении DINOv2-эмбеддингов
кадров и оценке трёх метрик: сходства стыка, амплитуды движения и гладкости
стыка.

## Состав репозитория

Проект разделён на два независимых подмодуля, каждый со своим `README`,
`pyproject.toml` и историей разработки:

- [`proto/`](./proto) — схемы сообщений `protobuf`, описывающие контракты
  взаимодействия между сервисами, и сгенерированный из них Python-пакет
  `proto-worker-service`. Это единый источник правды для форматов задач,
  ответов и сопутствующих структур.
- [`worker-service/`](./worker-service) — сервис-исполнитель, запускающий
  AI-модели и собирающий из них пайплайн генерации. Поддерживает два режима
  работы: одиночное приложение (локальный запуск) и сервис в составе более
  крупной системы (Kafka + S3 + Loki + Prometheus).

В работе также описывается отдельный сервис-менеджер, отвечающий за
оркестрацию пайплайнов и общение с пользователем. В составе настоящего
репозитория его исходный код не публикуется, однако сервис-исполнитель
спроектирован таким образом, чтобы быть готовым к работе под управлением
такого менеджера.

## Архитектура в двух словах

Система построена по микросервисной схеме (см. рис. 2.1б в ВКР). Менеджер
публикует задачи в брокер сообщений Kafka, исполнители подписываются на
интересующие их топики и возвращают результаты тем же путём. Крупные бинарные
артефакты — изображения, наборы кадров и атласы — передаются через
S3-совместимое объектное хранилище: в брокер кладутся только пути до них.
Контракты сообщений описаны в `proto/` и используются всеми участниками
взаимодействия.

Такое разделение позволяет независимо масштабировать различные части системы:
выделять больше исполнителей под генерацию видео или меньше — под
постобработку, а также при необходимости заменять файловое хранилище на
распределённое без правок бизнес-логики.

## Как воспроизвести результаты

Минимальный сценарий — запуск генерации анимации одного персонажа в режиме
одиночного приложения. Для этого достаточно собрать `worker-service` и
передать ему конфигурацию пайплайна:

```sh
cd worker-service
uv sync --extra standalone
worker-standalone -c config/config.yml config/pipeline.looped.yml \
    -- --inputs.first_frame=assets/inputs/girl.png
```

Подробные инструкции по обоим режимам, формату YAML-конфигураций и сборке
Docker-образов приведены в [`worker-service/README.md`](./worker-service/README.md).

## Используемые модели

| Этап | Модель | Лицензия |
| --- | --- | --- |
| Генерация когерентных изображений | Qwen-Image-Edit-2509 | Apache 2.0 |
| Генерация видеофрагментов | Wan2.2 | Apache 2.0 |
| Удаление фона | InspyreNet | открытая |
| Карты нормалей | StableNormal | открытая |
| Эмбеддинги для поиска цикла | DINOv2 | открытая |

Все модели выбраны с учётом ограничений на открытость весов, лицензирование
и размер: они могут быть запущены локально на потребительских видеокартах без
обращения к внешним API-сервисам.
